# Code Of Conduct

This project is for building reliable, secure, voice-first meeting automation. Contributors are expected to keep discussion technical, direct, and respectful.

## Expected Behavior

- Be clear and specific when reporting bugs or proposing changes.
- Critique code, architecture, tests, and documentation without personal attacks.
- Respect security boundaries and do not post secrets, private meeting content, or customer data.
- Assume good intent, but ask for evidence when technical claims matter.
- Keep contribution discussions focused on making the project safer, easier to operate, and easier to extend.

## Unacceptable Behavior

- Harassment, threats, discrimination, or personal insults.
- Publishing secrets, tokens, private keys, or private meeting data.
- Attempting to bypass project security controls.
- Submitting intentionally deceptive code, tests, prompts, or documentation.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests; restrict participation; or report abuse to the platform when behavior puts people, systems, or data at risk.
