# auto-bot v2.0 — Agent-Native Work Surface

*A board where agents are first-class citizens, MCP is the universal external surface, and every mutation flows through a single audited path with dry-run, undo, and a kill switch on top.*

v2.0 turns auto-bot from a voice-driven Scrum recorder into an agent-native work
surface. The board, the meeting, and the MCP server are all views on the same
canonical state. Agents propose; humans confirm; the system shows you the
diff first, lets you undo with one click, and can pause every long-running
agent in the org with a single button.

This release lands on `agent-first-v2-sprint-0` (HEAD `75f737eb`), 62 commits
ahead of `main`, +26,394 / -748 LOC across 188 files.

## What's new

### Sprint 0 — Foundations

Sprint 0 carved the v1 monolith into the package shape that the rest of v2.0
needed. Nothing in the user-visible product changed, but the seams that made
every later sprint possible were cut here.

- **`internal/*` package extraction with sealed import boundaries.** `cdc0e0e6` extracted `internal/board` domain types. `1fb5c865` extracted `internal/agent` run types. `0f5bfafe` moved the `agentRun` struct and methods into `internal/agent`. `86825e20` extracted `internal/meetings` for scrum + transcript types. After Sprint 0, `cmd/server` wires concrete types and HTTP surface; everything else lives behind a `internal/*` import path that the linter enforces.
- **Multi-tenant scoping.** `1dc77810` threaded `TenantID` through `auth`, `board`, `agent` runs, and the SQLite store. Every read and write now takes a tenant in scope; cross-tenant access requires explicit elevation and is logged. The hosted control plane in Sprint 5 hangs off this scope.
- **React + Vite + TypeScript SPA at `/app/`.** `e7e2c906` scaffolded the frontend with the Observatory Deck dark palette (`#0F1419` ground, `#1E40AF` action, `#10B981` accent). Strict TypeScript, ESLint, Prettier, and a 250 KB gzipped bundle budget. CI gate added in `2464f08f` enforces lint + test + bundle-size on every PR.
- **CI, agent controls, and product proof metrics.** `2464f08f` lit up the GitHub Actions pipeline that runs lint, vet, race-detector tests, frontend tests, and pre-commit hooks on every push. The same workflow uploads coverage and bundle-size reports.

### Sprint 1 — Agents as first-class citizens

Sprint 1 is the philosophical core of v2.0. The board no longer assumes the
assignee on a card is a human. It uses a discriminated union — `Actor` — and
agents get a full lifecycle (Plan → Run → Checkpoint → AskHuman → Resume →
Cancel) that's durable, observable, and pausable.

- **`board.Actor` discriminated type.** `043ca73b` introduced `Actor` with `Kind ∈ {human, agent}`, `ID`, `DisplayName`, and an optional `RunID` for agent actors. Card assignees are now `Actor`; Reporter and Watchers remain `User` (humans only). The `UnmarshalJSON` accepts the legacy `User` shape and promotes it to `Actor{Kind: "human"}`, so v1 snapshots load cleanly.
- **`agent.Run` extended with Plan, Cost, WaitingOn, SequenceNumber.** `4b07f1cc` added the `Plan` field (ordered list of steps with rationale + risk classification), `Cost` (tokens + dollar accumulator with a per-Run cap), `WaitingOn` (nil, or a `RunQuestionID` if the Run is parked on a human answer), and `SequenceNumber` (monotonic per-Run write counter for optimistic concurrency on Checkpoint).
- **`RunCoordinator` interface.** `bcb50084` introduced the interface that every agent goes through: `Start(ctx, RunSpec) (Run, error)`, `Checkpoint(ctx, runID, state) error`, `AskHuman(ctx, runID, question) (RunQuestion, error)`, `Resume(ctx, runID, answer) error`, `Cancel(ctx, runID, reason) error`. The voice meeting agent, the MCP tools, and the future hosted control plane all consume the same interface.
- **`RunStore` interface for persistence.** Same commit added `RunStore` with SQLite tables `run_checkpoints` (run_id, sequence_number, state_json, created_at) and `run_questions` (id, run_id, prompt, suggested_answers_json, asked_at, expires_at, answered_at, answer_json, state). Questions have a TTL; expired questions cannot be answered (see DA-1 fix below).
- **ULID-based question IDs.** `bcb50084` chose ULID over UUID4 for `RunQuestionID` so the IDs sort chronologically — debugging a stuck Run is dramatically easier when `LIST WHERE state='asked' ORDER BY id` matches the actual ask order.
- **WebSocket broadcasts.** `9aa953d4` broadcasts `run_question_asked`, `run_question_answered`, `run_question_expired`, `run_paused`, `run_resumed`, and the full Plan delta over the same `/ws` connection the React UI already uses. Tenant-scoped fanout (see SA-1 fix in the security section).

### Sprint 2 — MCP server

Sprint 2 made auto-bot reachable from any MCP-capable client. The MCP server
is not a side channel; it routes every mutation through the same dispatch
path the voice meeting uses, which means ActionLedger, risk classification,
and confirmation gates all apply uniformly.

- **`cmd/mcpd` binary with stdio + HTTP transports.** `5c8ccedd` shipped the
  MCP server foundation: `cmd/mcpd/main.go` boots either stdio (for Claude
  Desktop / VS Code / Cline) or HTTP (for hosted clients), with bearer-token
  auth gating the HTTP transport via `internal/mcp/auth.go`. A
  `docker-compose.yml` service and `Dockerfile.mcpd` ship alongside.
- **Five tools.** `board.list_cards`, `board.get_card`, `card.create`,
  `card.update`, `card.comment`. The first two are read-only; the last
  three are mutations.
- **S2.1 — wired through `/internal/tools/dispatch`.** `0b5df0dd` is the
  commit that matters most architecturally. Before this commit, the MCP
  tools spoke to a stub board; after it, they speak to the same
  `ApplyToolCall` dispatcher that the voice agent uses. ActionLedger
  records who did what and why; the risk classifier decides whether a tool
  needs human confirmation; the confirmation gate works for MCP exactly
  like it works for voice. There is no "MCP bypass" path.
- **API reference.** `4793d1f0` (despite its swapped title — see erratum) shipped 988 LOC of React; the actual API reference (`docs/api/mcp-tools.md` +648, `docs/api/openapi.yaml` +322) landed in `d1034796`. Both files are on the branch.

### Sprint 3 — Canonical board + projections

Sprint 3 separated the board's canonical model from the external trackers it
syncs with. Jira used to be hardcoded into the dispatcher; now it's one
projection among (eventually) many.

- **`projection.Projection` interface.** `d0904b60` introduced
  `internal/projection/projection.go` with the three required methods —
  `Push(ctx, Card) error`, `Pull(ctx) ([]Card, error)`, `Map(externalID) (CardID, error)` — plus a `Capabilities()` method that declares which fields the projection supports round-tripping.
- **`internal/projection/jira` is the canonical example.** Same commit moved the existing Jira sync into the projection shape and registered it via the registry in `extensions.go`.
- **Contract test in `internal/projection/contracttest`.** Same commit added a generic suite that any projection implementation can `t.Run` against. The Jira projection passes; future Linear + GitHub Issues projections will be measured against it.
- **(Pending S3.1.)** Linear + GitHub Issues projections are scoped for the next sprint. The interface is stable.

### Sprint 4 — Trust ceremony + closed-loop standup

Sprint 4 added the operator-facing surface that makes it safe to actually
let agents make changes. Every recent mutation has a 1-click undo; every
risky mutation can be staged for review; the entire agent fleet can be
paused with one button.

- **Dry-run mode via `tenant_settings` + `pending_actions` table.** `6997cd58` added the per-tenant `dry_run_default` flag and the staging table. When dry-run is on, mutations land in `pending_actions` instead of executing; a human reviews the diff and approves or discards.
- **Diff preview endpoint.** `fae7cd34` added `GET /internal/pending/:id/diff` returning the before/after JSON for a staged action, plus a WebSocket `pending_action_staged` event so the React queue updates live.
- **1-click undo on every recent mutation.** `76fb8d98` surfaced an undo button in the `RecentMutations` component. The button calls `Connector.Undo(toolCallID)` on the connector that originally executed the mutation. Undo has been part of `Connector` since v1; Sprint 4 just made it discoverable.
- **Pause-all-agents kill switch.** `3c72637c` added `POST /internal/agents/pause-all` which sets a tenant-scoped `agents_paused_at` row; the `RunCoordinator` checks this on every `Checkpoint` and `Resume` and refuses to advance any Run while it's set. Unpause is a separate, explicit action that records who released the pause.
- **Pre-meeting agenda builder (`internal/standup.BuildAgenda`).** `4074724e` builds a structured agenda from open cards, pending questions, and stale assignments — the same shape the Nova Sonic agent expects.
- **Agenda injection at meeting start.** `096bc5a8` modified Nova Sonic's `sendInitSequence` to feed the built agenda as the meeting's opening context. The agent walks the agenda; the team walks with it.
- **Post-meeting closer (`standup.Close`).** `3c4884a3` consumes the meeting transcript at adjournment, extracts follow-up commitments, creates cards for them, and kicks Runs against any commitment that maps to an agent.
- **Cron scheduler.** `7635fa6c` runs `BuildAgenda` periodically and sweeps expired RunQuestions; the same scheduler will host the cost-meter sweep in Sprint 5.

### Async intake (Daria persona finding)

The Daria persona walkthrough (`docs/persona/daria.md`, committed via `8414d1d3`) surfaced that meetings are not the only intake surface — async teams need a way to submit work without joining a voice call. Sprint 4 added two:

- **`POST /intake/standup` accepts JSON or `text/plain`.** `4d8ef097` exposes the endpoint. JSON gets a strict schema; plain text gets parsed by the same Nova Sonic prompts used in meetings. Both feed `internal/intake` (`c1ca8c0e`) which calls `ApplyToolCall` exactly like a voice tool — risk gates, confirmation, ActionLedger all apply.
- **`POST /intake/slack` with HMAC signature verification.** `d225f6e1` is the Slack webhook adapter. It verifies `X-Slack-Signature` against the configured signing secret with a 5-minute timestamp window. Slack body → intake event → dispatcher → board.
- **React `IntakeForm` component.** `0643afa6` added the form to the SPA; it posts to `/intake/standup` and shows the resulting card link.
- **Self-assignment skips confirmation; cross-user requires it.** `f8586814` is the SA-2 alignment: if the caller's identity matches the assignee on the proposed card, the confirmation gate is bypassed (the user is acting on themselves). If the assignee is a different user, the gate fires. See the security section.

## Security fixes

Seven critical findings from the bug-wave audit (rolled up under merge `309fd680`), plus two earlier SA-1 / SA-2 fixes:

- **F1 — Empty-JSON Actor rejected in `UnmarshalJSON`.** `2cf92cf8`. An `Actor` deserialized from `{}` previously produced a zero-value record that the dispatcher treated as "system." Now `UnmarshalJSON` rejects empty / kindless inputs with a typed error.
- **F2 — Fail-loud on Checkpoint audit miss.** `33797c1e`. If the audit ledger write failed during a Checkpoint, the Run previously continued anyway. Now the error propagates and the Run is parked in `state='audit_failed'` until an operator intervenes.
- **F3 — `persistAgentRun` errors propagated to coordinator callers.** `b68b3124`. Silent persistence failures were eaten by a `_ = err` ignore. Restored as a returned error; `RunCoordinator.Start` now fails loudly if the store rejects the write.
- **F6 — MCP sentinel errors mapped to JSON-RPC `-32602`.** `874f160b`. `ErrRunNotFound`, `ErrRunQuestionNotFound`, and `ErrRunQuestionExpired` now surface as `-32602` (Invalid params) to MCP clients instead of bare `-32603` (Internal error), which lets clients distinguish "the run doesn't exist" from "the server is broken."
- **DA-1 — Resume blocked on expired questions.** `11cc11ca`. Previously, a Run could be resumed by answering a `RunQuestion` whose TTL had already lapsed — the zombie-Resume class of bug. Now `LoadRunQuestion` returns `ErrRunQuestionExpired` and `Resume` rejects the answer with a typed error.
- **`gosec` G706 annotations on `mcpd` log.Printf sites.** `95367dc2`. Lint unblocker; no behavior change.
- **SA-1 — Cross-tenant WebSocket fanout.** `50217b68`. The WebSocket hub previously broadcast every event to every connection. Now every event carries a `TenantID` and the hub filters by the subscriber's authenticated tenant. Without this, MCP-driven card edits on tenant A would have leaked to tenant B's open browser tab.
- **SA-2 — Confirmation gate keyed on tool risk, not dispatcher label.** `baa5c69a`. The confirmation gate previously checked `meta.Source != "voice"` as a proxy for "needs confirmation." MCP calls with empty Source bypassed the gate entirely. Now the gate keys on the tool's declared risk classification; every dispatch through `ApplyToolCall` hits the gate if the tool requires it. `meta.SkipConfirmation` is the only intentional bypass and requires a justification string.
- **SecArch-002 alignment for intake.** `f8586814`. Self-assignment skips confirmation; cross-user assignment requires it. Aligned with the SA-2 finding above.

## Architecture decisions

Four ADRs landed (or were ratified) this release:

- **ADR 0001 — Core Extension Boundaries.** `docs/adrs/0001-core-extension-boundaries.md`. Status: Accepted. Establishes the `internal/*` sealed-import rule and the `Connector` extension point as the only place new external systems should hook in.
- **ADR 0002 — Canonical Board With External Projections.** `docs/adrs/0002-canonical-board-with-external-projections.md`. Status: Proposed (Sprint 3 deliverable). Declares the canonical model lives in auto-bot's store; trackers like Jira, Linear, and GitHub Issues are projections that round-trip a declared subset of fields.
- **ADR 0003 — MCP Server as Universal External Surface.** `docs/adrs/0003-mcp-server-as-universal-external-surface.md`. Status: Proposed (Sprint 2 deliverable). Declares MCP as the primary external mutation surface; HTTP API is a thin adapter; voice is one client among many.
- **ADR 0004 — Multi-Tenant Model.** `docs/adrs/0004-multi-tenant-model.md`. Status: Proposed (Sprint 0 first slice, Sprint 5 hosted control plane). Declares TenantID as the single scope token threaded through auth, storage, and fanout.

## Deprecations / breaking changes

- **BREAKING — `MCPD_TOKEN` env var removed (#58 hard cut).** The MCP HTTP transport no longer accepts a single static bearer. Operators must set `MCP_SIGNING_KEYS` (kid:base64-32-byte-key) on both `cmd/server` and `cmd/mcpd`, then mint per-client tokens via `POST /admin/mcp-tokens` (HMAC-SHA256, scoped, ULID jti, 15-minute default TTL). cmd/mcpd refuses to serve the HTTP transport when `MCP_SIGNING_KEYS` is missing — fail-closed. See [docs/api/mcp-tools.md#authentication](../api/mcp-tools.md#authentication) for the full token model + rotation procedure. Migration: drop `MCPD_TOKEN` from your env, set `MCP_SIGNING_KEYS="k1:$(openssl rand -base64 32)"`, redeploy, then issue tokens for each MCP client.
- **`runs.start` MCP tool added (#59).** New tool routes through cmd/server's `assign_ticket_to_agent` so MCP clients (Claude Code, Cursor, agent SDK scripts) can mint agent Runs through the same audited path voice uses. Because the underlying tool is medium-risk, the default response is `202 Accepted` carrying a Trust Ceremony confirmation envelope; the Dry-Run Queue operator approves before the Run starts.
- **`internal/jira/` package removed (#60).** Empty stub with no imports. The Jira projection lived at `internal/projection/jira/` since Sprint 3; the placeholder package was misleading drift.
- **`meta.Source` field renamed to `meta.Dispatcher`** on the internal `toolCallMeta` struct. Internal Go type; not exposed to any API. The S2.1 dispatch envelope's public JSON name is `dispatcher`.
- **Confirmation gate no longer depends on `Source`.** Every dispatch through `ApplyToolCall` hits the confirmation queue if the tool requires it (SA-2 fix, `baa5c69a`). Callers that relied on bypassing confirmation by leaving `Source` empty — there should not be any in the tree, but third-party patches may have — must now set `meta.SkipConfirmation = true` explicitly with a justification string.
- **`RunStore.LoadRun` / `LoadRunQuestion` return `(value, error)`** with sentinel errors (`ErrRunNotFound`, `ErrRunQuestionNotFound`, `ErrRunQuestionExpired`) instead of the v1-style `(value, bool, error)` triple. Callers should use `errors.Is(err, runstore.ErrRunNotFound)` rather than checking a `found` boolean.
- **`internal/board.User` is still the type for `Reporter` + `Watchers`; `internal/board.Actor` is now the type for `Assignee`.** Pre-Actor snapshots load cleanly via `Actor.UnmarshalJSON`'s legacy-shape promotion (`043ca73b`). External consumers that round-trip Card JSON should be prepared for `assignee` to be either the legacy `{user fields...}` shape or the new `{kind, id, display_name, run_id?}` shape; serialization always emits the new shape.

## Known limitations (NOT shipped in v2.0, on roadmap)

- **S3.1 — Linear + GitHub Issues projections.** Interface (`projection.Projection`) is stable and contract-tested via `internal/projection/contracttest`. Jira is the only reference implementation in this release.
- **S5 — Cost meter + hosted control plane.** The `agent.Run.Cost` field is populated and the per-Run cap is enforced; what's missing is the tenant-level rollup, the billing-export path, and the hosted control plane that lets external operators see across tenants.
- **F1.2 — Server-side dispatch routes for `run.*` / `agent.*` tools.** The React UI sends these (e.g. take-over, answer, cancel from the CardRunTab); the server currently returns `400` and the UI surfaces the error inline. The dispatch route lands in F1.2.
- **Per-tenant card-ID counters can collide across tenants.** Card IDs are tenant-scoped reads but the shared sequence isn't yet tenant-partitioned. Fixed before the hosted control plane lights up; today's single-tenant deploys are unaffected.
- **Slack webhook tenant binding is hardcoded to `"default"`.** `/intake/slack` (`d225f6e1`) drops every event into the `default` tenant. Resolves with the hosted control plane in Sprint 5, which will map signing-secret → tenant.
- **6 commits on this branch have swapped or duplicate titles** from the 2026-05-26 racing-tree incident. Code is correct; only commit-message metadata is wrong. Documented in `docs/erratum-commit-title-swaps.md` with the full title → actual-content mapping. We chose not to rebase because rewriting 6 message bodies would change every downstream SHA reference (including `docs/security/silent-failure-scan-001.md`'s "Scope: commits 0f5bfaf..9aa953d" line) and force every existing reviewer to reset.
